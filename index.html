How x402 Payments Work

HYDRA uses the x402 HTTP payment protocol — a standard for machine-readable micropayments. No subscriptions. No API keys. No friction.

01

Call the Endpoint

Make a standard HTTP request to any paid endpoint — for example, POST /v1/markets/alpha. No credentials required for the initial call.

02

Receive Payment Details

The API responds with HTTP 402 Payment Required and headers specifying the exact USDC amount, wallet address, and Base chain ID. Your bot or client reads these automatically.

03

Pay and Resend

Send the specified USDC to 0x2F12A73e1e08F3BCE12212005cCaBE2ACEf87141 on Base. Include the transaction hash in the X-Payment-Proof header and resend your request — you get the full response instantly.

# Step 1: call the endpoint curl -X POST https://hydra-api-nlnj.onrender.com/v1/markets/alpha # → 402 Payment Required # X-Payment-Amount: 10.00 # X-Payment-Address: 0x2F12A73e1e08F3BCE12212005cCaBE2ACEf87141 # X-Payment-Network: base # X-Payment-Chain-Id: 8453 # Step 3: pay USDC on Base, then resend with proof curl -X POST https://hydra-api-nlnj.onrender.com/v1/markets/alpha \ -H "X-Payment-Proof: 0xYOUR_TX_HASH" \ -H "Content-Type: application/json" \ -d '{"context": "FOMC decision in 48h, SEC crypto enforcement expected"}'
